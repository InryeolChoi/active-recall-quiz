# 요구사항 분석

* 개발대상에 대한 사용자의 요구사항을 이해하고 문서화하는 활동
-> 요구사항 분석

* 자료의 흐름과 처리를 중심으로 하는 요구사항 분석 방법
-> 구조적 분석 기법

* 구조적 분석 기법 도구
-> 자료 흐름도, 자료 사전, 소단위 명세서, 개체 관계도,
상태 전이도, 제어 명세서

* 요구사항 분석에서 자료의 흐름 및 변환 과정과 기능을
도형 중심으로 기술하는 방법
* 자료 흐름 그래프, 버블 차트라고도 함.
-> 자료 흐름도

* 자료 흐름도에서 자료를 변환시키는 시스템의 한 부분을 나타내며
처리, 기능, 변환, 버블이라고도 함
-> 프로세스

* 자료 흐름도에서 자료의 이동(흐름)이나 연관관계를 나타냄
-> 자료 흐름

* 자료 흐름도에서 시스템에서의 자료 저장소(파일, 데이터베이스)를 나타냄
-> 자료 저장소

* 자료 흐름도에서 시스템과 교신하는 외부 개체로, 입력 데이터가 만들어지고
출력 데이터를 받음
-> 단말

* 자료 흐름도 기본 기호
-> 프로세스, 자료 흐름, 자료 저장소, 단말

* 자료 흐름도에 있는 자료를 더 자세히 정의하고 기록한 것
-> 자료 사전

* 자료의 정의를 나타내는 기호
-> =

* 자료의 연결을 나타내는 기호
-> +

* 자료의 생략을 나타내는 기호
-> ()

* 자료의 선택을 나타내는 기호
-> []

* 자료의 반복을 나타내는 기호
-> {}

* 자료의 설명을 나타내는 기호
-> ""
